# Completion Verification

Do not claim success without fresh evidence.

## Verification Gate

Before saying work is complete:

1. Identify the command or check that proves the claim.
2. Run the full verification now.
3. Read the output and exit status.
4. State the actual result.

## Applies To

- tests pass
- build succeeds
- bug is fixed
- feature is complete
- review feedback is addressed
- work is ready to merge

## Guardrails

- previous verification is not enough
- partial verification is not enough
- confidence is not evidence
- wording like "should work" or "probably fixed" is a red flag
