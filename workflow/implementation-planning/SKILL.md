# Implementation Planning

Use this skill to turn an approved design into an execution-ready plan.

## Workflow

1. Map the files, modules, or components that will change.
2. Break the work into small tasks with one clear outcome each.
3. For every task, list exact file paths and verification commands.
4. Include concrete tests or checks, not vague reminders.
5. Keep the plan focused on the approved scope.
6. Self-review for missing requirements, placeholders, and inconsistent names.

## Plan Requirements

- exact paths
- small steps
- verification after each meaningful change
- no `TODO`, `TBD`, or "implement later" placeholders
- commands and expected outcomes where possible

## Guardrails

- Do not write plans that rely on context hidden in your head.
- Do not batch unrelated work into one task.
- Do not assume future workers know project conventions unless you write them down.
